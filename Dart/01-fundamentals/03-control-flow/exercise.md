# Exercise - Control Flow and Decision Making in Dart

## Part A - Core Implementation
Write a script that evaluates a list of tasks or orders and assigns statuses based on priority, completeness, and due conditions.

## Part B - Validation and Error Cases
Handle missing values, empty lists, and invalid thresholds without producing misleading output.

## Part C - Reliability and Refactor
Refactor branching logic into small helper functions with names that explain the business rules.

## Constraints
- Use guard clauses for invalid input.
- Keep nesting depth low.
- Test boundary cases explicitly.

## Acceptance Criteria
- Status assignment is correct for normal and edge cases.
- Rules are readable and easy to modify.
- Empty or invalid input is handled safely.

## Bonus Challenge
Add a `switch`-based categorization step for a second dimension such as region or task type.

## Reflection Prompts
- Which condition became confusing first?
- What refactor made the decision logic easier to trust?
