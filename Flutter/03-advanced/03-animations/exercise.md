# Exercise - Flutter Animations and Motion Design

## Part A - Core Implementation
Add one or two purposeful animations to a screen such as item insertion, screen transition, or validation feedback.

## Part B - Validation and Error Cases
Ensure the UI still works correctly if the animation is interrupted, reversed, or triggered repeatedly.

## Part C - Reliability and Refactor
Refactor animation code so state changes and motion concerns remain understandable.

## Constraints
- Only animate where it improves comprehension.
- Keep animation timing consistent and restrained.
- Avoid introducing jank on common devices.

## Acceptance Criteria
- The motion feels intentional.
- Repeated interaction does not break the UI.
- Animation code remains maintainable.

## Bonus Challenge
Add a reduced-motion fallback or at least document where it would belong.

## Reflection Prompts
- Which animation improved UX most clearly?
- Where would more motion have made the app worse?
