# Exercise - Flutter Setup and Developer Workflow

## Part A - Core Implementation
Install Flutter, create a fresh project, and run it successfully on at least one emulator or physical device.

## Part B - Validation and Error Cases
Resolve any missing SDK, Android toolchain, or device-connection issues and record the fixes.

## Part C - Reliability and Refactor
Document a repeatable local workflow for creating, running, and rebuilding Flutter apps.

## Constraints
- Do not mark setup complete until diagnostic warnings are reviewed.
- Test at least one change through hot reload.
- Record exact commands that worked in your environment.

## Acceptance Criteria
- The app launches locally.
- Common setup failures are understood, not just bypassed.
- You have a written workflow you can repeat later.

## Bonus Challenge
Run the same starter app on both an emulator and a physical device.

## Reflection Prompts
- Which setup issue consumed the most time?
- What would you automate or document better next time?
