# Exercise - Flutter Widgets Basics

## Part A - Core Implementation
Build a small screen with text, buttons, icons, cards, and one reusable custom widget.

## Part B - Validation and Error Cases
Handle empty or missing display data gracefully instead of rendering broken UI.

## Part C - Reliability and Refactor
Extract repeated UI into named widgets and simplify any oversized `build()` methods.

## Constraints
- Keep each custom widget focused.
- Do not let one screen own every visual element inline.
- Prefer readable composition over clever nesting.

## Acceptance Criteria
- The UI renders correctly.
- Repeated elements are extracted meaningfully.
- Empty-state rendering is considered.

## Bonus Challenge
Add a toggle or local interaction that requires a small stateful widget.

## Reflection Prompts
- Which widget extraction improved readability most?
- Where did local state naturally belong?
