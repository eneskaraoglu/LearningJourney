# Exercise - Flutter Forms and Validation

## Part A - Core Implementation
Build a form with at least three fields, validation rules, and a submit action.

## Part B - Validation and Error Cases
Handle empty fields, malformed values, and duplicate or invalid submissions clearly.

## Part C - Reliability and Refactor
Extract validation helpers and simplify the submit flow so each failure path is understandable.

## Constraints
- Do not allow invalid submissions.
- Show actionable error messages.
- Keep controller lifecycle and cleanup correct.

## Acceptance Criteria
- Field validation works.
- Submit flow reflects loading, success, and failure states.
- The form code remains readable after adding rules.

## Bonus Challenge
Add cross-field validation such as password confirmation or date range checks.

## Reflection Prompts
- Which validation rule was easiest to mis-handle?
- What refactor made the submit flow safer?
