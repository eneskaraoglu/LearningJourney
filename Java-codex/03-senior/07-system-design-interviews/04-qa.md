# Q&A

Q: What is the first step in design interviews?
A: Clarify requirements and constraints.

Q: Why define the API early?
A: It frames data flow and contract expectations.

Q: How do you estimate scale?
A: Use user count, QPS, and data size assumptions.

Q: What is a key tradeoff in caching?
A: Freshness versus speed.

Q: When use message queues?
A: For async workflows and buffering spikes.

Q: What is consistency vs availability?
A: Strong consistency may reduce availability in partitions.

Q: How do you show reliability?
A: Use redundancy, monitoring, and graceful degradation.

Q: What is a common interview pitfall?
A: Jumping to architecture before clarifying requirements.
