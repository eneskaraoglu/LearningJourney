# Q&A

Q: What is a bounded context?
A: A boundary where a domain model and language are consistent.

Q: What is an aggregate?
A: A cluster of entities with a root enforcing invariants.

Q: Why use hexagonal architecture?
A: It keeps the domain independent of external systems.

Q: What is an anemic domain model?
A: Entities with data only and logic pushed into services.

Q: Where do domain events live?
A: Inside the domain layer, raised by aggregates.

Q: What is a value object?
A: An immutable object identified by its value, not an ID.

Q: Why separate application services?
A: They orchestrate use cases without containing domain rules.

Q: How does DDD help interviews?
A: It shows clear reasoning about boundaries and consistency.
