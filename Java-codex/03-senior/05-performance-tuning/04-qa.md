# Q&A

Q: What is tail latency?
A: High percentile response times like p95 and p99.

Q: Why focus on database indexes?
A: They often dominate performance in CRUD systems.

Q: What is a slow query plan?
A: The database chooses inefficient scanning or joins.

Q: How do you size a connection pool?
A: Based on CPU, DB limits, and expected concurrency.

Q: Why is batching useful?
A: It reduces network and transaction overhead.

Q: What is the risk of over-parallelism?
A: Context switching and resource contention.

Q: What is a performance budget?
A: A target for latency and throughput constraints.

Q: When is caching the best optimization?
A: For read-heavy data with acceptable staleness.
