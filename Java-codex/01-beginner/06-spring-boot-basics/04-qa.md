# Q&A

Q: What does `@SpringBootApplication` include?
A: `@Configuration`, `@EnableAutoConfiguration`, and `@ComponentScan`.

Q: Why prefer constructor injection?
A: It makes dependencies explicit and supports immutability and tests.

Q: What is a `@RestController`?
A: A controller that returns JSON instead of views.

Q: Why use DTOs?
A: To control the API contract and avoid exposing internal fields.

Q: How do profiles work?
A: Boot loads profile-specific config based on `spring.profiles.active`.

Q: What is auto-configuration?
A: Boot configures beans based on classpath and properties.

Q: How do you handle validation errors?
A: Use `@Valid` and a global exception handler.

Q: What is the role of `application.yml`?
A: Centralized configuration for the app and its dependencies.
