# Q&A

Q: `Runnable` vs `Callable`?
A: `Callable` returns a value and can throw checked exceptions.

Q: Why use `ExecutorService`?
A: It manages threads, reuses them, and simplifies task submission.

Q: What is a stream pipeline?
A: A chain of intermediate operations ending with a terminal operation.

Q: What does `collect` do?
A: It gathers stream results into a container like a list or set.

Q: When should you avoid parallel streams?
A: When tasks are small or when order and thread safety are concerns.

Q: What is a race condition?
A: Multiple threads access shared state without proper synchronization.

Q: What is `Future`?
A: A handle to a pending result of an asynchronous computation.

Q: Why are streams lazy?
A: Intermediate operations run only when a terminal operation executes.
