# Q&A

Q: What is a slice test?
A: A focused Spring test that loads only part of the context.

Q: Why avoid heavy `@SpringBootTest` usage?
A: It is slower and harder to isolate failures.

Q: How do you override properties in tests?
A: Use `@TestPropertySource` or `DynamicPropertySource`.

Q: What is `@MockBean`?
A: A Spring Boot annotation that replaces a bean with a Mockito mock.

Q: When should you use Testcontainers?
A: When you need real infrastructure like databases or Kafka.

Q: What does `@DataJpaTest` configure?
A: Repository beans and JPA-related components only.

Q: How do you test validation?
A: Use MockMvc and assert on error responses.

Q: Why use configuration properties?
A: Type safety and centralized configuration.
