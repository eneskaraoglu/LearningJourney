# Q&A

Q: Why use messaging?
A: Decouple services, improve reliability, and handle bursts.

Q: What is idempotency?
A: Processing the same message multiple times without side effects.

Q: What is a dead-letter queue?
A: A place for messages that fail repeatedly.

Q: Cache-aside pattern?
A: App checks cache first, then loads from DB and populates cache.

Q: What is cache invalidation?
A: Removing or updating stale entries after writes.

Q: When is caching risky?
A: When data is highly volatile or strict consistency is required.

Q: Why is the outbox pattern useful?
A: It keeps DB changes and events consistent.

Q: Kafka vs RabbitMQ?
A: Kafka is log-based and high throughput; RabbitMQ is broker-queue oriented.
