# Q&A

Q: What does `volatile` guarantee?
A: Visibility of changes across threads, not atomicity.

Q: `synchronized` vs `ReentrantLock`?
A: `ReentrantLock` offers tryLock and timed locking with more control.

Q: What is a deadlock?
A: Threads waiting on each other?s locks indefinitely.

Q: What is a race condition?
A: Concurrent access that produces inconsistent results.

Q: When to use `ConcurrentHashMap`?
A: When many threads read and write shared maps safely.

Q: What is the fork-join framework?
A: A work-stealing pool for divide-and-conquer tasks.

Q: What is `CompletableFuture` used for?
A: Async pipelines with composition and error handling.

Q: Why avoid shared mutable state?
A: It is the main source of concurrency bugs.
